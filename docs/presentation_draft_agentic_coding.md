# 発表資料ドラフト：Agentic Coding 〜Antigravityを使ってPoC開発してみた〜

## 概要
Google DeepMindによるAIエージェント「Antigravity」を活用し、従来の開発フローとは一線を画す「Agentic Coding」によるPoC開発の実践報告。

## セッション構成（50分）

### 1. イントロダクション (5分)
- **Agentic Codingの定義**: 
  - 単なる補完 (Copilot) から「自律的な遂行 (Agent)」へ。
  - AIが環境（ターミナル、ファイル、Git）を自ら操作する次世代の開発。
- **Antigravityのポテンシャル**:
  - 文脈理解、計画立案、実行、自己修正のサイクル。

### 2. 開発プロセスの実態 (10分)
- **タスクの構造化 (`task.md`)**:
  - 巨大な要件をAIがいかに「実行可能な最小単位」に分解するか。
- **合意形成 (`Implementation Plan`)**:
  - 実装前に人間が方針を確認する、AIエージェントとの「ペアプロ」スタイル。
- **実演（ログ・履歴の活用）**:
  - AIが自らエラーを検知し、シェルを叩いて修正する泥臭くも強力なプロセス。

### 3. 今回のPoC：AI見積もりシステム V2 (20分)
- **ユースケース**: チャットベースのデザイン・開発見積もり相談アシスタント。
- **SIer基準のRAG構築**:
  - 800人規模のSIer相場をRAGで学習させ、精緻な見積もりを出力。
- **UXの徹底的な磨き込み**:
  - **Contextual Suggestion**: 自由記述から機能を動的に提案。
  - **Multi-select UI**: チャットにおける入力負荷の低減。
  - **Immersive Design**: 徹底的な無駄の排除による没入感。

### 4. 知見：AIエージェントとどう働くか (10分)
- **プロンプトではなく「コンテキスト」**:
  - 具体的で詳細な指示がAIの「意思決定」をどう変えるか。
- **開発効率の正体**:
  - コードを書くスピードではなく、「実装方針の検証と適用」の高速化。
- **エンジニアに求められる役割の変化**:
  - 手を動かす人から、AIの計画をレビューし、アーキテクチャを統制する「監督者」へ。

### 5. 結論とQ&A (5分)
- 「AIがコードを書く」の先にある、ソフトウェア開発の新しい風景。

---

## デモで見せるべきポイント
1. AIが `task.md` を更新しながら進捗を管理している様子。
2. AIが作成した `Implementation Plan` の詳細さ。
3. 今回実装した「複数機能選択UI」などの動的に生成されたコード部分。

# Agentic Coding Demonstration Guide (for Non-Engineers)

非エンジニアの同僚の方に「エージェントコーディング（Antigravity）」を説明・実演するためのガイドです。

## 1. 説明のコンセプト： 「超優秀な新人プログラマとのペア」
難しい技術用語は使わず、以下のように例えるのがスムーズです。

- **従来のチャットAI (ChatGPT等)**: 「物知りな相談相手」。アドバイスはくれるが、実際の作業は人間がやる。
- **エージェントコーディング (Antigravity)**: 「指示を理解して動く新人エンジニア」。指示を出すと、自分でファイルを読み、コードを書き、テストまでやって報告してくれる。

- **「クラウド・ネイティブ」の提示**: 「この画面は私の手元のPCで動いていますが、AIの『頭脳』はクラウド（Azure）上にあります。これによって、世界中のどこからでも同じ知能にアクセスできるんです」といった、システムとしての広がりを強調。

## 3. 【一押し】視覚的なインパクト： ブラウザ操作のデモ
非エンジニアの方に最も驚かれるのが、AIが「実際に画面を見て、操作する」様子です。

- **実演内容**: 「Azureポータルを開いて、現在のデプロイ状況を確認して」や「特定のサイトの構成を調べて」と依頼する。
- **見どころ**: 
    - AIがブラウザを起動し、ページをスクロールしたり、ボタンを探したりする様子。
    - **「DOM（サイトの裏側の構造）を読み取っています」**という実況。
    - 人間と同じように「画面を見て」判断していることが視覚的に伝わります。

## 4. おすすめの実演シナリオ
短時間で「おおっ」と思ってもらえる、具体的で見た目に変化のある作業がおすすめです。

### シナリオ例： 「ドキュメントの整理と追加」
1. **依頼**: 「新しく参加するメンバーのために、プロジェクトの概要を1枚のドキュメントにまとめて。既存の `docs/` 内のファイルを参考にしてね」
2. **AIの動きを見せる**:
    - AIが既存の `v2_technical_stack.md` や `rag_implementation.md` を読みに行く。
    - AIが内容を要約し、新しいファイル案を作成する。
    - **「自分で読んで考えている」**ことを強調。
3. **修正依頼**: 「少し内容が難しいから、もっと平易な言葉にして」と追加で頼む。
    - 対話によって内容が磨かれていく様子を見せる。

### シナリオ例： 「UIの微調整」（コードがわかる場合）
1. **依頼**: 「フロントエンドのボタンの色をもう少し親しみやすいオレンジに変更して。あと、マウスを乗せた時に少し大きく動くようにして」
2. **AIの動きを見せる**:
    - CSSファイルを見つけ出す様子。
    - 具体的なコードを書き換える様子。
    - ブラウザでの確認（もし可能なら）。

## 4. 伝えたいメッセージ
実演の最後に、以下を伝えると「自分たちの仕事にどう関わるか」が伝わります。

- 「人間は『何を作りたいか（WHAT）』を考え、AIが『どう作るか（HOW）』を担当する」
- 「単純作業や調査をAIに任せることで、人間はより創造的な設計や意思決定に集中できるようになる」

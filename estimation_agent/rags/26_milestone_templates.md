## 1. マイルストーンとは

### 1.1 定義
マイルストーンとは、プロジェクトの重要な節目となるイベントや成果物の完成時点を指します。

### 1.2 目的
- プロジェクトの進捗を可視化
- ステークホルダーとの合意ポイントを明確化
- リスクの早期発見
- チームのモチベーション維持

### 1.3 設定基準
- **測定可能**: 完了/未完了が明確に判断できる
- **重要性**: プロジェクトの成否に影響する
- **タイミング**: 適切な間隔で設定（2-4週間ごと）

## 3. マイルストーン管理のベストプラクティス

### 3.1 マイルストーン設定時

**適切な粒度:**
- 大きすぎる: 進捗が見えにくい
- 小さすぎる: 管理コストが増大
- 推奨: 2-4週間に1つのマイルストーン

**明確な完了条件:**
- 曖昧な表現を避ける
- 測定可能な基準を設定
- チェックリスト形式で定義

**ステークホルダーとの合意:**
- マイルストーンの重要性を共有
- 完了条件の合意
- レビュー・承認プロセスの明確化

### 3.2 マイルストーン実行時

**定期的な確認:**
- 週次で進捗確認
- マイルストーン到達予定日の2週間前に最終確認
- 遅延リスクの早期検知

**ドキュメント管理:**
- 成果物の品質確認
- レビュー記録の保管
- 承認証跡の保管

**コミュニケーション:**
- マイルストーン達成時の報告
- 遅延発生時の早期エスカレーション
- ステークホルダーへの定期報告

### 3.3 マイルストーン遅延時の対応

**影響分析:**
1. 遅延の原因特定
2. 後続マイルストーンへの影響評価
3. プロジェクト全体への影響評価

**対策立案:**
1. リソース追加
2. スコープ調整
3. スケジュール調整
4. リスク対策の実施

**ステークホルダー報告:**
1. 遅延の事実報告
2. 原因と影響の説明
3. 対策案の提示
4. 承認取得

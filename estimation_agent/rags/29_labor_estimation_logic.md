# 概算工数積算ロジック (Revised 2026.01)

本ドキュメントは、AIがプロジェクトの概算工数（人日）を算出するための最終決定ロジックです。
見積もり・提案書の生成、および根拠の提示には必ず本ロジックを使用してください。

---

## 1. フェーズ別積算ルール

### Phase 2: 設計フェーズ (SIer内製)
内製チームが実装のための仕様を固めるフェーズです。
- **共通設計**: 3人日 (IA/サイトマップ/全体フロー)
- **画面詳細設計**:
    - 標準画面: 0.5人日 / 1画面 (ワイヤーフレーム作成含む)
    - 複雑な画面（ダッシュボード等）: 0.8人日 / 1画面
- **機能仕様化**: 
    - 選択された「機能」1つにつき 0.5人日 を加算 (開発用IF定義等)

### Phase 3: ビジュアルフェーズ (デザイン外注)
デザイン会社へ委託する、プロフェッショナルなUI作成フェーズです。
**積算 = [A] 共通基盤 + [B] 機能別キー画面設計 + [C] 画面展開工数**

- **[A] 共通基盤 (必須)**:
    - デザインシステム / コンポーネント定義: 5人日 (小規模は3人日)
    - プロトタイプ作成: 2人日
- **[B] 機能別キー画面設計 (機能ベース)**:
    - 複雑な機能（決済、データ可視化、管理画面）: 1.5人日 / 1キー画面
    - 標準的な機能（認証、一覧・詳細、投稿）: 1.0人日 / 1キー画面
- **[C] 画面展開工数 (画面数ベース)**:
    - 全画面数からキー画面（機能数分）を引いた残り: 0.5人日 / 1画面

### 開発フェーズ (SIer内製)
**積算 = Σ(機能別工数) + (画面数ベース開発工数)**

- **機能別工数**: `pricing_matrix.md` の「機能別工数目安」を参照。
- **画面数ベース開発工数**: 
    - 1.5人日 / 1画面 (平均)
    - 複雑度調整: 簡易×0.8、標準×1.0、高難度×1.3

---

## 2. 諸経費・バッファ

- **PM / ディレクション費**: (Phase2 + Phase3 + 開発) の合計工数の **15%**
- **テスト / QA費**: 開発工数の **20%**
- **外注管理費 (Phase 3用)**: Phase 3費用の **15%**

---

## 3. 単価設定

- **SIer単価 (Phase 2, 開発)**: ￥50,000 / 人日
- **デザイン外注単価 (Phase 3)**: ￥80,000 / 人日

---

## 4. 計算の優先順位

1. ユーザーから「画面数」の指定がある場合は、必ず **[C] 画面展開工数** を計算に含める。
2. 機能が複数選択されている場合、それぞれの機能に基づいた **[B] キー画面設計** を計上する。
3. ユーザーの「デザイン準備状況（Readiness A~D）」に応じて、Phase 2, 3 の実施有無を判断する。
    - **A (アイデアのみ)**: Phase 2, 3 ともにフルで計上。
    - **B (仕様書あり)**: Phase 2 を 50% に削減、Phase 3 はフル。
    - **C (構成案あり)**: Phase 2 不要、Phase 3 フル。
    - **D (デザイン済み)**: Phase 2, 3 ともに不要。

---
**Data Source**: 経営企画部・開発推進室 (2026.01 更新版)

## 5. 不確実性とリスク変動幅 (Variance)

Phase 3 (デザイン外注) における見積額は、要件の確定度（Confidence）に応じて以下の変動幅（Variance）を持ちます。これは外注先の作業手戻りリスクや仕様変更リスクを統計的に見込んだものです。

| 確定度 (Confidence) | 変動幅 (Variance) | 理由・根拠 |
| :--- | :--- | :--- |
| **Low (要件がまだ曖昧)** | **±40%** | 具体的なデザインイメージが未定であり、複数回の修正・提案ラリーが発生する可能性が高いため。 |
| **Medium (標準的)** | **±20%** | ラフイメージ等は存在するが、詳細なトーン＆マナーやコンポーネント定義が未確定であるため、標準的なリスクを見込む。 |
| **High (仕様確定済み)** | **±10%** | ブランドガイドラインや類似画面が存在し、制作物が明確に定義されているため、リスクは最小限に留まる。 |

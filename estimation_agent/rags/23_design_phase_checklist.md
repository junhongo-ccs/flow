# デザインフェーズ完全チェックリスト

営業担当者がデザイン会社とスムーズに協業するための完全チェックリスト

---

## Phase 1: 準備（営業担当）

### 要件整理
- [ ] プロジェクト目的の明確化
- [ ] ターゲットユーザーの定義（年齢・職業・利用シーン）
- [ ] 必要な機能のリストアップ
- [ ] 参考サイトの収集（3-5サイト）
- [ ] 予算の確定
- [ ] スケジュールの確定
- [ ] ブランドガイドラインの確認（あれば）

### ワイヤーフレーム作成
- [ ] 画面一覧の作成
- [ ] 画面遷移図の作成
- [ ] 各画面のワイヤーフレーム作成
- [ ] レスポンシブ対応の考慮（PC/タブレット/スマホ）
- [ ] 社内レビュー完了

---

## Phase 2: デザイン会社選定

### 候補選定
- [ ] 3社以上から見積もり取得
- [ ] ポートフォリオ確認（類似案件の実績）
- [ ] Figma使用可否確認
- [ ] 納期確認
- [ ] コミュニケーション方法確認（Slack, メール等）

### 契約前確認
- [ ] 修正回数の確認（通常2-3回）
- [ ] 納品形式の確認（Figma, Sketch, Adobe XD）
- [ ] 著作権の確認
- [ ] 追加費用が発生するケースの確認
- [ ] 開発者への引き継ぎサポート有無

### 契約
- [ ] 契約書の作成・締結
- [ ] 着手金の支払い
- [ ] キックオフ日程の調整

## Phase 3: デザイン制作

### キックオフミーティング
- [ ] プロジェクト概要の共有
- [ ] ターゲットユーザーの説明
- [ ] ワイヤーフレームの共有
- [ ] 参考サイトの共有
- [ ] デザインの方向性確認
- [ ] スケジュール確認
- [ ] コミュニケーション方法の確認

### コンセプト提案レビュー
- [ ] デザインの方向性確認（2-3案）
- [ ] フィードバック提出（3営業日以内）
- [ ] 方向性の決定

### デザイン初稿レビュー
- [ ] 主要画面のデザイン確認
- [ ] ブランドイメージとの整合性確認
- [ ] ターゲットユーザーへの適合性確認
- [ ] フィードバック提出（3営業日以内）

### 修正版レビュー（1回目）
- [ ] フィードバックの反映確認
- [ ] 追加修正点の洗い出し
- [ ] フィードバック提出（2営業日以内）

### 修正版レビュー（2回目）
- [ ] 最終調整の確認
- [ ] 全画面の整合性確認
- [ ] 承認

### 最終確認
- [ ] 全画面のデザイン確認
- [ ] レスポンシブ対応の確認
- [ ] ホバー・フォーカス状態の確認
- [ ] エラー状態の確認
- [ ] アニメーション・インタラクションの確認

## トラブル対応チェックリスト

### デザインが要件と合わない場合
- [ ] 具体的な相違点の洗い出し
- [ ] デザイン会社への説明
- [ ] 修正範囲の確認（契約内か追加費用か）
- [ ] 修正スケジュールの調整

### 開発時に実装困難が判明した場合
- [ ] 実装困難な理由の明確化
- [ ] デザイン会社への相談
- [ ] 代替案の検討
- [ ] クライアントへの説明・承認

### スケジュール遅延が発生した場合
- [ ] 遅延理由の確認
- [ ] 影響範囲の確認
- [ ] リカバリープランの作成
- [ ] クライアントへの報告

---

## 成功のためのヒント

### コミュニケーション
- デザイン会社への質問は具体的に
- フィードバックは建設的に
- レスポンスは迅速に（24時間以内）

### 準備
- ワイヤーフレームは必ず作成
- 参考サイトは複数用意
- 予算とスケジュールは余裕を持って

### レビュー
- 感覚的な意見ではなく、根拠を持って
- ターゲットユーザーの視点で評価
- 開発者の意見も取り入れる
